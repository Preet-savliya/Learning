Polymorphism - Polymorphism means one thing many forms. in C# Polymorphism allows the same method name to work in different ways.

Types of Polymorphism in C#
---- Compile time Polymorphism (Method Overloading) : same method name but different parameters

class MathOperation
{
    public int Add(int a, int b)
    {
        return a + b;
    }

    public int Add(int a, int b, int c)
    {
        return a + b + c;
    }
}

âœ” method chosen at compile time

---- Run-time Polymorphism (Method Overriding) : Child class changes the behaviour of parent class method. same method but different parameters

class Animal
{
    public virtual void Sound()
    {
        Console.WriteLine("Animal makes a sound");
    }
}

class Dog : Animal
{
    public override void Sound()
    {
        Console.WriteLine("Dog barks");
    }
}
âœ” Method chosen at run time

Usage :

Animal a = new Dog();
a.Sound();   // Output: Dog barks

Why Use Polymorphism ?
---- Makes code flexible
---- Easy to extend
---- Reduces complexity

Real-life Example :-

1. Remote button
---- Same button
---- Different TV ---> Different Behaviour

2. Vehicle Example
Parent class : Vehicle
Child class : Bike, Car, Bus.

Method StartEngine()
---- Bike : engine starts with the key.
---- Car : engine start with the button.
---- Bus : engine start with the igniation lever.
