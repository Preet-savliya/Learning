1. What is GIT ?
--- Git is a distributed version control system that tracks changes in source code, allowing multiple developers to work on the same project safely.

Why Git is Used ?
-- Track Code changes 
-- Work in teams 
-- Roll back to previous versions
-- Maintain code history 
-- Support CI/CD pipelines


2. Git vs GitHub
| Git                  | GitHub              |
| -------------------- | ------------------- |
| Version control tool | Hosting platform    |
| Runs locally         | Runs on cloud       |
| Tracks changes       | Stores repositories |
| CLI-based            | Web-based UI        |

Git = tool
GitHub / GitLab / Bitbucket = platforms


3. Installing Git
Windows 
-- Download from: https://git-scm.com
-- Verify:
      git --version


4. Git Basic Concepts
| Term              | Meaning                  |
| ----------------- | ------------------------ |
| Repository (repo) | Project tracked by Git   |
| Commit            | Snapshot of changes      |
| Branch            | Parallel version of code |
| Merge             | Combine branches         |
| Remote            | Online repository        |
| Clone             | Copy repo from remote    |
| Push              | Send changes to remote   |
| Pull              | Get changes from remote  |



5. Create your First Git Repository

Step 1: Initialize Git

-- git init

Step 2: Check Status 

-- git status


6. Staging & Committing Changes
Working Tree → Staging → Commit

git add .
git commit -m "Initial commit"

| Command    | Purpose       |
| ---------- | ------------- |
| git add    | Stage changes |
| git commit | Save snapshot |

8. Connecting to GitHub

Step 1: Create Repo on Github 
--- Repository Name
--- Do Not select README
--- Copy repository URL

Step 2: Add Remote Origin 
Code:
git remote add origin https://github.com/username/repo-name.git

Verify:
git remote -v


Step 3: Push Code to GitHub

git branch -M main 
git push -u origin main



9. Cloning a Repository

git clone https://github.com/username/repo-name.git

10. Branching in git

git branch feature-login 

Switch Branch 

git checkout feature-login

OR

git switch feature-login

Create + Switch 

git checkout -b feature-login




1️⃣1️⃣ Merging Branches

git checkout main
git merge feature-login


1️⃣2️⃣ Pulling Latest Changes

git pull origin main

1️⃣3️⃣ Pushing Changes

git add .
git commit -m "Added login feature"
git push origin feature-login

1️⃣4️⃣ Pull Requests (PR)

1. Push branch to GitHub
2. Click Compare & Pull Request
3. Add description
4. Click Merge

✔ Used in team collaboration


1️⃣5️⃣ Viewing History

git log

One line view:

git log --oneline

1️⃣6️⃣ Undoing Changes

Undo Working Directory Change

git checkout -- filename

Unstage File

git reset filename

Undo Last Commit (keep changes)

git reset --soft HEAD~1

Undo Last Commit (delete changes)

git reset --hard HEAD~1

⚠ Careful with --hard

1️⃣7️⃣ Stashing Changes

git stash
git stash list
git stash apply


1️⃣8️⃣ Delete Branch

Local
git branch -d feature-login

Remote
git push origin --delete feature-login

1️⃣9️⃣ Git Configuration
git config --global user.name "Your Name"
git config --global user.email "you@email.com"

View config:
git config --list
